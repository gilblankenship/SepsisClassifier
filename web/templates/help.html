Introduction

SepsisDx is a machine learning system for detecting sepsis from urine ELISA biomarker measurements. It uses two urinary biomarker ratios — CRP/Creatinine and IP-10/Creatinine — to classify patients as septic or non-septic.

The system is validated against the blood-based gold standard: positive blood culture combined with Sepsis-3 criteria (SOFA score ≥ 2).

Why Urine?

Traditional sepsis detection relies on blood draws and lab cultures, which are invasive and slow. Urine-based ELISA testing offers a non-invasive, rapid alternative. This software improves on the existing rule-based SepsisDx classifier by using machine learning to catch borderline cases that hard-threshold rules miss.

Biomarkers

Biomarker Unit SepsisDx Threshold Description
CRP / Creatinine pg/mg > 300 → Septic C-Reactive Protein normalized by urinary creatinine
IP-10 / Creatinine pg/mg > 100 → Septic Interferon gamma-induced Protein 10 (CXCL10) normalized by creatinine

Values are derived from urine ELISA assays. Creatinine normalization corrects for urine dilution.

Classifiers

SepsisDx Baseline

Rule-Based

CRP/Cr > 300 OR IP-10/Cr > 100 → Septic. Hard thresholds miss borderline cases like Sample 6 (CRP/Cr=299, IP-10/Cr=80).

Logistic Regression

ML

L2-regularized with internal feature scaling. Fast, well-calibrated probabilities. Good baseline ML model.

Random Forest

Ensemble

200 decision trees (max depth 5). Handles non-linear boundaries. Typically the best overall performer.

XGBoost

Gradient Boosting

200 estimators with automatic class imbalance handling. Uses all 7 features (basic + engineered).

Neural Network

Deep Learning

MLP architecture: 32 → 16 → 8 → output. ReLU activation, Adam optimizer, early stopping, minority class oversampling. Best calibration (lowest ECE).

Output columns added to your CSV:

  • ml_prediction — "Septic" or "Not Septic"
  • ml_risk_level — LOW / MODERATE / HIGH / VERY HIGH
  • sepsisdx_prediction — rule-based system comparison
  • models_agree — whether ML and SepsisDx agree

Training Models

The training pipeline runs the following steps:

  1. Data Loading — loads the selected dataset
  2. Cross-Validation — stratified 5-fold CV for all selected classifiers
  3. Threshold Optimization — finds optimal classification thresholds
  4. Model Export — trains on full dataset, exports as .pkl with JSON metadata

Note: Training runs in the background. You can monitor progress in real time via the progress page. Training on 5,000 samples typically takes 30–60 seconds.

SHAP Explainability

SHAP (SHapley Additive exPlanations) values show how each biomarker contributes to individual predictions. This is critical for clinical trust — clinicians need to understand why a model makes a prediction.

Positive SHAP Value

Feature pushes prediction toward "Septic"

Negative SHAP Value

Feature pushes prediction toward "Not Septic"

Example: Borderline Sample 6 (Septic)

CRP/Cr = 299.0, IP-10/Cr = 80.0

  • IP-10/Cr = 80.0 → +0.5922 (increases sepsis risk)
  • CRP/Cr = 299.0 → -0.0475 (slightly decreases risk)

The ML model learned that IP-10/Cr = 80 is clinically significant, unlike the hard threshold rule (which requires > 100).

Threshold Optimization

The default threshold of 0.5 is not always optimal for clinical use. The system evaluates four strategies:

Strategy Goal Best For
max_f1Maximize F1 scoreBalanced precision/sensitivity
max_sensitivity_90specMax sensitivity, specificity ≥ 90%Screening (ED triage)
max_youdenMaximize Youden's JOptimal balance (default)
min_missedMinimize false negativesCritical care: never miss sepsis
